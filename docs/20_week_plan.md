# 松果 20 周详细执行计划

## 阶段 1：定义与最小大脑

### 第 1 周：产品定义与架构

目标：明确松果第一版到底做什么，不做什么。

学习：

- Agent、Memory、Tools、Avatar 的基本概念。
- 什么是 MVP。
- 软件原型和硬件原型的先后关系。

执行：

- 写产品 MVP 文档。
- 写角色圣经 v1。
- 画系统架构。
- 明确第一版只做三个核心问题：全球关键事件、天气穿衣、Outlook 日程。

交付：

- `docs/product_mvp.md`
- `docs/role_bible_v1.md`
- `docs/system_architecture.md`

### 第 2 周：开发环境与项目骨架

目标：让项目可以运行。

学习：

- VS Code 基本使用。
- Python 虚拟环境。
- Git 基础。
- `.env` 配置文件。

执行：

- 创建后端目录。
- 建立 Python 虚拟环境。
- 写第一个 `hello_sunguo.py`。
- 准备 API Key 配置方式。

交付：

- 可运行的 Python 项目骨架。
- 一条命令能启动松果 v0。

### 第 3 周：第一个聊天管家

目标：松果能用稳定人格聊天。

学习：

- Prompt。
- System message。
- 对话历史。
- API 调用。

执行：

- 把角色圣经转成 system prompt。
- 实现命令行聊天。
- 保存最近 20 轮对话。
- 测试松果是否保持温柔、阳光、聪明的风格。

交付：

- 命令行聊天 Demo。
- `prompts/sunguo_system_prompt.md`

### 第 4 周：早安汇报 v0

目标：先不用真实 API，也能生成固定结构的早安汇报。

学习：

- JSON。
- 函数。
- 模板化输出。

执行：

- 设计早报结构。
- 用假数据生成一份早报。
- 让松果用自然语言播报早报。

交付：

- `morning_brief_demo.py`
- 一份样例早安汇报。

## 阶段 2：真实数据接入

### 第 5 周：天气模块

目标：接入真实天气或先接入可替换的天气适配器。

学习：

- API 请求。
- 城市、温度、天气、降雨概率。
- 异常处理。

执行：

- 写天气工具接口。
- 生成穿衣建议。
- 处理 API 失败时的 fallback。

交付：

- 天气 + 穿衣建议 Demo。

### 第 6 周：全球关键事件模块 v1

目标：做出昨天全球关键事件摘要。

学习：

- 新闻摘要。
- 信息分类。
- 来源和时间戳。
- 不确定性表达。

执行：

- 先手动输入新闻文本或链接。
- 按经济、政治、股市、科技、地缘风险分类。
- 输出 5-8 条关键事件。

交付：

- 全球关键事件摘要 Demo。

### 第 7 周：Outlook 日程接入方案

目标：明确 Outlook 如何接入，并先跑通测试数据。

学习：

- Microsoft Graph 基础概念。
- OAuth 授权。
- Calendar / To Do 数据模型。

执行：

- 写 Outlook 接入方案文档。
- 如果授权条件具备，尝试读取真实日历。
- 如果暂时不具备，用本地 mock 数据模拟。

交付：

- Outlook 接入方案。
- 今日事项汇总 Demo。

### 第 8 周：早安汇报 v1

目标：把全球事件、天气、Outlook 三个模块合成一个完整早报。

学习：

- 数据聚合。
- 信息排序。
- 重点提炼。

执行：

- 汇总三个模块。
- 生成“最重要的三件事”。
- 输出松果口吻的早安播报。

交付：

- 完整早安汇报 v1。

## 阶段 3：Agent 能力与主动提醒

### 第 9 周：待办系统

目标：松果能管理本地待办。

学习：

- SQLite。
- CRUD。
- 状态字段。

执行：

- 新增、查看、完成、删除待办。

交付：

- 自然语言管理待办 Demo。

### 第 10 周：提醒系统

目标：松果可以主动提醒。

学习：

- 定时任务。
- 时间解析。
- 提醒状态。

执行：

- 支持固定时间提醒。
- 支持会议前提醒。

交付：

- 到点提醒 Demo。

### 第 11 周：记忆系统 v1

目标：松果能记住用户偏好，但可控可删。

学习：

- 用户画像。
- 记忆类型。
- 敏感信息。

执行：

- 建立 profile、memories 表。
- 用户确认后才写入长期记忆。
- 支持查看和删除记忆。

交付：

- 可管理记忆系统。

### 第 12 周：Agent 工具调用

目标：松果能判断什么时候该聊天、查天气、看日程、建待办。

学习：

- Function calling / tool calling。
- 工具路由。
- 失败处理。

执行：

- 给 Agent 接入天气、早报、待办、日程工具。
- 加入执行确认。
- 记录工具调用日志。

交付：

- Agent 工具调用 Demo。
