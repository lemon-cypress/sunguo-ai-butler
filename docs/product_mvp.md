# 松果 AI 私人管家 MVP

## 1. 项目定位

松果是一个具备数字人形象的 AI 私人管家。第一阶段目标不是立刻完成真实全息硬件，而是先做出一个可运行、可演示、可持续迭代的 Character-to-Agent 原型。

第一版松果应当像一个温柔、阳光、聪明的 18 岁女生管家。她能在每天早上主动汇报：

- 前一天全球关键事件：经济、政治、股市、科技、地缘风险等。
- 当天天气、温度、降雨、空气质量、穿衣建议。
- Outlook 日历和待办事项：今天要做什么、优先级、提醒事项。

长期目标是把同一套 Agent 大脑迁移到实体底座、画框屏、伪全息显示、裸眼 3D 显示，以及未来更复杂的空间显示终端。

## 2. 第 1 阶段 MVP 范围

### 必做

- 稳定人格的 AI 管家对话。
- 每日早报生成。
- 全球关键事件摘要。
- 天气与穿衣建议。
- Outlook 日历读取与日程总结。
- 本地配置、日志和基础隐私边界。
- 后续可接数字人的结构化输出协议。

### 暂不做

- 真实全息光学模组。
- 支付、门锁、自动发消息等高风险执行。
- 自动读取未授权私有群消息。
- 复杂智能家居全屋控制。
- 自研大模型。
- 角色商城和多人家庭权限。

## 3. 第一版核心用户场景

### 早安汇报

用户早上启动松果，松果用温柔自然的口吻汇报：

1. 昨天全球发生了哪些关键事件。
2. 这些事件对经济、市场和个人决策有什么影响。
3. 今天本地天气如何，应该怎么穿。
4. Outlook 中今天有哪些会议、任务和注意事项。
5. 今天最值得优先完成的 3 件事。

### 当日提醒

松果根据 Outlook 日程和本地待办，在关键时间提醒用户：

- 会议前提醒。
- 重要任务截止前提醒。
- 药物、运动、休息等固定提醒。

### 后续数字人呈现

松果的回复不仅是文字，还会包含结构化状态：

```json
{
  "dialogue": "早上好，今天上海有小雨，建议穿薄外套并带伞。",
  "emotion": "warm",
  "action": "morning_briefing",
  "gesture": "gentle_wave",
  "location": "desk"
}
```

这些字段未来会驱动 Unreal 中的数字人动作、表情、语音和位置。

## 4. 成功标准

20 周后，松果至少应达到：

- 每天能生成一份完整早报。
- 能接入真实或测试 Outlook 日历。
- 能根据天气给出穿衣建议。
- 能生成全球关键事件摘要。
- 能用统一人格进行语音或文字播报。
- 能以 JSON 协议驱动数字人演示。
- 有一个 90 秒产品 Demo。
- 有一份硬件 PRD v0.1。
